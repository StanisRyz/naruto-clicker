# Manual Backend Cloud Sync — Validation Checklist (C5.1)

## Static validation

```bash
godot --headless --editor --quit
godot --headless --script res://scripts/tools/GenerateLocalizationData.gd
godot --headless --script res://scripts/tools/ValidateLocalizationDataFreshness.gd
godot --headless --script res://scripts/tools/ValidateLocalizationExport.gd
git status
git diff --stat
```

All localization scripts must exit 0. No unintended files should appear in `git status`.

> **Note:** Godot headless cannot run in this Windows dev environment without a display
> driver. Run these commands on a Linux CI runner or inside a Godot headless container.
> The `LocalizationData.gd` file was regenerated by the editor plugin automatically.

## Android Guest mode

| Step | Expected |
|---|---|
| Cold-start with no account session (or logout first) | AuthGate shows; choose "Continue as guest" |
| Open Settings → Account section | Cloud save controls are **hidden**; status shows "Sign in to use cloud save" |
| Verify no network requests are sent | No `PUT /v1/save` or `GET /v1/save` traffic in logcat |
| Local save works normally | Progress saves and loads without touching backend |

---

## Android Account mode — Save to Cloud

| Step | Expected |
|---|---|
| Log in with a valid test account | AuthGate completes; gameplay starts |
| Open Settings → Account section | Cloud save controls visible: "Save to Cloud" and "Load from Cloud" buttons |
| Make some progress (buy hero level) | Local state changes |
| Press **Save to Cloud** | Status changes to "Uploading cloud save..." immediately |
| Wait for response | Status changes to "Cloud save uploaded" on success |
| Inspect YDB `cloud_saves` table | Row for `user_id` updated with new JSON blob |
| Press **Save to Cloud** again when upload fails (disable wifi) | Status shows backend error code; no crash |

---

## Android Account mode — Load from Cloud

| Step | Expected |
|---|---|
| Log in with a test account that has a cloud save | Cloud save controls visible |
| Press **Load from Cloud** | Inline warning appears: "Loading from cloud will replace local progress." |
| Press **Cancel** | Warning disappears; no network request; local save unchanged |
| Press **Load from Cloud** again → **Confirm Load** | Status: "Loading cloud save..."; network request sent |
| Wait for response — cloud save exists | Status: "Cloud save loaded"; gameplay UI reflects loaded progress |
| Verify local save file updated | `user://save_v1.json` contains the cloud data |
| Load from cloud when `has_save=false` | Status: "No cloud save found"; local save unchanged |
| Load invalid/corrupted cloud save | Status: "Cloud save is invalid"; local save unchanged |
| Network error during download | Status shows backend error code; no crash; local save unchanged |

## Local save integrity

| Step | Expected |
|---|---|
| Old local save (pre-C5.1) loads correctly | No regression; save migration handles missing `cloud_save_meta` |
| Local save is NOT deleted before a successful cloud load | `apply_cloud_save_payload` only writes on validation success |
| Local save is NOT modified on cloud load failure | Failure returns `false` without touching `save_v1.json` |
| Reset Progress still works after cloud load | Normal reset flow unaffected |
| Prestige still works after cloud load | Normal prestige flow unaffected |

## Log hygiene

| Step | Expected |
|---|---|
| Upload save | No full JSON payload printed to logcat |
| Download save | No full JSON payload printed; no session tokens; no passwords |
| Error paths | Only error code / operation name logged; no sensitive data |
